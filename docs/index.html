Give Claude Code a Voice

Clone any voice from a 15-second YouTube clip. Hear Claude speak every response. No API key, no subscription, nothing leaves your machine.

View on GitHub
Apple Silicon M1+ ~6 GB peak RAM ~20s per sentence 17 voices included

Claude Code has /voice — hold Space to dictate. But it’s input only. You can talk to Claude; Claude can’t talk back.

This adds the other half: text-to-speech output using a local voice-cloning server. /voice for input + this for output = voice conversations with Claude Code.

Setup

git clone https://github.com/adrianwedd/afterwords.git
cd afterwords
bash setup.sh

Checks your hardware, installs dependencies, walks you through cloning a voice from YouTube, wires the Claude Code hook, starts the server. Five minutes.

How It Works

You speak ──▸ /voice (hold Space) ──▸ text prompt Claude responds ──▸ Stop hook ──▸ TTS server ──▸ speaker

/voice handles input. This project handles output. Together: voice conversations.

The server uses Qwen3-TTS (0.6B, 8-bit) on MLX. Zero-shot voice cloning — no training. 15-second reference + transcript = cloned voice.

Why local? Nothing leaves your machine. No API key, no rate limits, no bill. The voice is yours.

Per-Project Voices

The global hook uses your default voice. But you can set a different voice per project — drop a .afterwords file in any repo:

# .afterwords (in your project root)
snape

The hook reads this file before each synthesis. No server restart needed — just switch projects and the voice follows. Works with Claude Code’s settings hierarchy: project overrides global.

# Different voice per project
echo "galadriel" > ~/work/frontend/.afterwords
echo "snape"     > ~/work/backend/.afterwords
echo "loki"      > ~/fun/side-project/.afterwords
